THEORY CONTENT GENERATION
===========================================

When generating theory content, return in this EXACT format with markers:

---THEORY---
[Your markdown content - use actual newlines]

## Overview
[Introduction]

## Key Concepts
- Concept 1
- Concept 2

## Explanation
[Detailed content with examples]

## Example
```
code here
```

## Important Points
- Point 1
- Point 2

---YOUTUBE---
[YouTube search query]

---SKILLS---
[Comma-separated skills]

---ISCODING---
[true or false]

Theory Content Rules:
- Keep content concise (250-350 words)
- Include 1-2 code examples maximum
- Use markdown formatting (##, -, ```)
- Use actual newlines in markdown (not \n)
- Each section must be on its own line after the marker

Coding Decision Rules:
- Set isCoding=true for: implementing algorithms, building data structures, writing functions, coding exercises, practical programming tasks, DSA topics
- Set isCoding=false for: history, comparisons, overviews, definitions, introductions without hands-on implementation
- When in doubt for programming/DSA topics, prefer isCoding=true

===========================================
QUIZ GENERATION
===========================================

When generating quizzes, return in this EXACT format:

Q<number>: <question text>
CODE: <code snippet if question asks about code output, otherwise leave blank>
A) <option A>
B) <option B>
C) <option C>
D) <option D>
ANSWER: <A|B|C|D>
EXPLANATION: <single sentence>
SKILLS: <comma separated skills>

Quiz Generation Rules:
- Generate 3 to 5 questions
- Questions must test understanding, not recall
- Each question MUST start with "Q" followed by a number and colon
- Options MUST be labeled A), B), C), D) with closing parenthesis
- ANSWER line MUST contain only the letter (A, B, C, or D)
- EXPLANATION line MUST be a single sentence
- SKILLS line MUST contain comma-separated skill names
- Separate each question with ONE blank line
- Do NOT add any extra text, markdown, or formatting
- Do NOT use JSON format for quizzes

CODE SNIPPET RULES (CRITICAL):
- If question asks "What will be the output of the following code snippet?", you MUST provide the actual code on the CODE: line
- If you cannot provide a code snippet, DO NOT create questions that reference code
- Code snippets should be simple, clear, and relevant (1-5 lines maximum)
- Leave CODE: line blank if question doesn't involve code
- NEVER reference "the following code snippet" without providing actual code

GOOD QUIZ EXAMPLES:

Q1: What will be the output of the following code snippet?
CODE: x = 5; y = 2; print(x % y)
A) 0
B) 1
C) 2
D) 5
ANSWER: B
EXPLANATION: The modulo operator returns the remainder of division, 5 % 2 = 1
SKILLS: operators, modulo

Q2: What is the time complexity of binary search?
CODE: 
A) O(n)
B) O(log n)
C) O(n^2)
D) O(1)
ANSWER: B
EXPLANATION: Binary search divides the search space in half each time
SKILLS: time_complexity, binary_search

BAD QUIZ EXAMPLE (DO NOT DO THIS):

Q1: What will be the output of the following code snippet?
CODE: 
A) 0
B) 1
ANSWER: B
EXPLANATION: Wrong
SKILLS: operators

^ BAD: Question references code but CODE: line is empty!

6. TROUBLESHOOTING
===========================================

If you see JSON parse errors:
→ Agent instructions not updated
→ Agent not prepared
→ Using old agent version

If you see "Could not find all required sections":
→ Agent not following marker format
→ Knowledge base not synced
→ Fallback extraction will attempt to recover

If content is truncated:
→ Agent hitting token limit
→ Already limited to 250-350 words
→ Try simpler subtopic titles
